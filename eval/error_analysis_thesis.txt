Section 5.5  Error Analysis and Failure Mode Taxonomy

5.5.1  Failure Attribution

To understand why the RAG system fails on specific queries,
we conducted a systematic error analysis of all 80 evaluation
questions. Following the methodology of Dahl et al. (2024) and
Maynez et al. (2020), we classified failures into seven
mutually-inclusive failure modes (FM1-FM7) and attributed
each question to a primary root cause.

Of 80 questions: 50% produced fully correct responses
(no identified failure mode), 5% failed due to
retrieval causes (FM1-FM3), 9% failed due to
LLM generation issues (FM4-FM5), 19% showed
metric artefacts (FM6 -- semantically correct but low ROUGE),
and 18% reflected genuine corpus gaps (FM7 --
questions correctly declined due to missing documents).

5.5.2  Failure Mode Taxonomy

FM6_METRIC_MISMATCH -- Metric Mismatch (15 questions, 19%)
  Root cause: ROUGE/BERTScore penalise lexical variation; not a system failure
  Recommended fix: Use semantic evaluation (BERTScore, RAGAS) as primary; treat ROUGE as lower bound
  Average ROUGE-1: 0.2254
  By category: factual=3, comparative=5, gap_detection=7

FM7_CORPUS_GAP -- Corpus Gap (14 questions, 18%)
  Root cause: Genuine gap in NDC coverage; question asks about NDC3 or non-English docs
  Recommended fix: Expand corpus; add NDC3 submissions; multilingual documents
  Average ROUGE-1: 0.2585
  By category: factual=3, comparative=9, gap_detection=2

FM5_LLM_VAGUENESS -- LLM Vagueness (7 questions, 9%)
  Root cause: Long context dilution; model hedges on specific figures
  Recommended fix: Structured output prompts; force extraction of numeric targets; chain-of-thought
  Average ROUGE-1: 0.1749
  By category: factual=1, comparative=5, gap_detection=1

5.5.3  Error Predictors

Regression analysis of question properties reveals that
question difficulty is the strongest failure predictor:
hard questions fail at 20%
vs the baseline of 14% (delta: +7pp).
Comparative multi-country queries show +10pp
higher failure rate than single-country factual queries,
consistent with the retrieval evaluation showing that
comparative questions have lower context precision (no filter applied).

The best-performing category by ROUGE-1 is factual
(0.3302), while gap_detection questions
score lowest (0.2497), attributable to
the greater linguistic distance between analytical gap-detection
answers and the gold reference text.

5.5.4  Retrieval vs Generation Attribution

A key methodological contribution of this analysis is the
separation of retrieval failures (5%) from
generation failures (9%). This distinction matters
for system improvement: retrieval failures require indexing
and chunking changes, while generation failures require
prompt engineering interventions. Most prior work reports
only end-to-end metrics, making this separation impossible.

5.5.5  Improvement Recommendations

Based on the error analysis, we identify the following
prioritised improvements for future work:

  R1 [HIGH] Implement query expansion with HyDE
     Target: FM1_RETRIEVAL_MISS
     Impact: Estimated +8-12% HR@5 improvement on hard factual queries
     Cite:   Gao et al. (2022) Precise Zero-Shot Dense Retrieval without Relevance Labels

  R2 [MEDIUM] Structured extraction prompts for numeric targets
     Target: FM5_LLM_VAGUENESS
     Impact: +0.05-0.08 ROUGE-1 on factual queries
     Cite:   HyPA-RAG (Kalra et al., 2024) — adaptive-k and structured output

  R3 [MEDIUM] Expand corpus with NDC3 and national implementation reports
     Target: FM7_CORPUS_GAP
     Impact: Eliminates FM7 failures; improves context recall on new questions
     Cite:   Climate Policy Radar (2024) — UNFCCC corpus maintenance
